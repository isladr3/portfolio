Professional Self-Assessment

Resourceful Development

I’ve always had an interest in computer science, but I finally had the opportunity to make it a reality in 2024. My approach to development is based on understanding things deeply. I don't stay in one lane; I'm the kind of developer who is resourceful and finds solutions anywhere. I look at technical problems the way Steve Jobs looked at calligraphy—gathering inspiration from different fields to build something that actually works. Coming from a background in journalism at the University of Utah and working in real estate, I’ve learned that the most effective tools often come from being "nosy" enough to read the entire operations manual and understand the system as a whole.

Collaboration and Communication

Software doesn't exist in a vacuum. My experience in journalism taught me how to communicate with people who have different needs, and I apply that to technical stakeholders. In code reviews, I’m focused on accountability. I don’t believe in "throwing code over the fence" and hoping a tester finds the holes. I want my code to be production-ready before anyone else sees it. Whether I’m documenting a complex logic flow or walking a team through a new feature, I keep things direct and technically sound without using jargon just for the sake of it.

Engineering and Data Strategy

For my capstone project, I refactored a mobile inventory app into a professional system called Equator. I moved the project to the MVVM architecture because it creates a clean separation between the UI and the data. It makes the app easier to maintain and debug. On the data side, I implemented Merge Sort and Binary Search to handle inventory lookups. This changed the app from a simple list to an efficient tool that handles data with O(log n) efficiency. I care about how the code is structured because that's what makes it scalable.

Security and Cloud Integration

I want my code to work and be resilient. Using Firebase Authentication for the Equator app allowed me to build a security-first foundation from day one. By moving the data layer from a local SQLite setup to a Firebase Cloud Database, I ensured that data is secure, tied to specific users, and synced in real-time. Security isn't an extra feature to me; it’s a requirement for building software that people can actually trust.

Moving Forward

The artifacts in this portfolio are proof of my ability to transition from a functional prototype to a professional-grade application. I’m ready to bring this resourceful, deep-dive mindset to a software development team where I can keep solving complex problems and building systems that deliver real value.
